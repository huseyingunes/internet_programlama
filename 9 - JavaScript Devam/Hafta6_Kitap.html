Bölüm 01

var, let ve const: Üç Anahtar Kelimenin Farkı

JavaScript'te değişken tanımlamanın üç yolu birbirinin alternatifi değildir — her birinin farklı kapsam kuralları, hoisting davranışı ve yeniden atama politikası vardır.

var — Fonksiyon Kapsamlı, Hoisted, Eski Usul

var JavaScript'in 1995'ten beri var olan orijinal anahtar kelimesidir. İki belirgin özelliği vardır: fonksiyon kapsamı ve hoisting.

JavaScript 
function ornek() {
  if (true) {
    var mesaj = "Merhaba";
  }
  console.log(mesaj); // "Merhaba" — if bloğu dışında!
}

for (var i = 0; i < 3; i++) { }
console.log(i); // 3 — döngü dışında bile erişilebilir!
⚠
Yaygın Hata

for döngüsünde var kullanmak, döngü değişkeninin döngü bittikten sonra da var olmaya devam etmesi anlamına gelir. Asenkron callback'ler (setTimeout, event listener) döngü bittikten sonra çalıştıklarında, hepsi döngünün son değerini okur — JavaScript'te en sık karşılaşılan hata kaynaklarından biridir.

let — Blok Kapsamlı Modern Standart

let ES6 (2015) ile eklendi ve var'ın fonksiyon kapsamı sorununu çözdü. Tanımlandığı blok içinde yaşar; blok dışına çıkmaz:

JavaScript 
function ornek() {
  if (true) {
    let mesaj = "Merhaba";
    console.log(mesaj); // "Merhaba" ✓
  }
  console.log(mesaj); // ReferenceError: mesaj is not defined
}

for (let i = 0; i < 3; i++) { }
console.log(i); // ReferenceError ✓ — döngü dışında yok

let x = 10;
x = 20; // ✓ geçerli — yeniden atama
let x = 30; // SyntaxError — yeniden tanım yasak

const — Sabit Referans

const değişkenin referansını kilitler, nesnenin içeriğini değil. Bu kritik bir ayrımdır:

JavaScript 
const PI = 3.14159;
PI = 3; // TypeError: Assignment to constant variable

const kullanici = { ad: "Ahmet", yas: 25 };
kullanici.yas = 26; // ✓ geçerli — nesne içeriği değişebilir
kullanici = {};     // TypeError — referans değiştirilemez

// Gerçekten dondurulmuş nesne için:
const SABIT = Object.freeze({ PI: 3.14159, E: 2.71828 });
✦
Altın Kural

Başlangıç noktası olarak const kullanın. Yeniden atama gerekiyorsa let'e geçin. var'ı hiçbir koşulda kullanmayın — modern JS kod tabanlarında var görmek kötü pratik işaretidir.

Gerçek Dünya Örnekleri

DOM Referansları

JavaScript

input.value = 'Yeni değer'; // ✓ içerik değişebilir
input = null;              // TypeError — referans değişemez

Sabit Yapılandırmalar

JavaScript 
const CONFIG = {
  apiUrl:     'https://api.example.com',
  timeout:    5000,
  maxRetries: 3,
};

CONFIG.timeout = 8000; // ✓ izin verilir
CONFIG = {};            // TypeError — referans değiştirilemez

var'ın Gerçek Tehlikesi: Asenkron Döngüler

var'ın en tehlikeli davranışı asenkron callback'lerle birleştiğinde ortaya çıkar:

JavaScript 
// var ile HATA — tüm butonlar 5 yazar!
for (var i = 0; i < 5; i++) {
  document.getElementById('btn' + i).addEventListener('click', function() {
    console.log('Buton ' + i + ' tıklandı');
  });
}
// Tüm butonlar "Buton 5 tıklandı" yazar

// let ile DOĞRU — her iterasyonda yeni kapsam
for (let i = 0; i < 5; i++) {
  document.getElementById('btn' + i).addEventListener('click', function() {
    console.log('Buton ' + i + ' tıklandı');
  });
}
// Her buton kendi i değerini yazar: 0, 1, 2, 3, 4

Hangisi Ne Zaman?

  • const — değer ömrü boyunca yeniden atanmayacaksa: konfigürasyon, DOM referansları, fonksiyonlar, sabitler.
  • let — sayaç, döngü değişkeni, koşullu olarak değişen değerler.
  • var — kullanmayın. Yeni kodda yeri yoktur.
ℹ
ESLint

ESLint veya TypeScript kullanan modern projelerde no-var kuralı varsayılan olarak açıktır; var kullanıldığında lint hatası verir.

Strict Mode ve Değişken Bildirimi

Strict modda bildirilmemiş değişkene atama yapmak artık hata üretir:

JavaScript 
// Normal mod — sessiz global yaratır:
function normal() {
  kazara = 42; // Global değişken oluşturur!
}

// Strict mode — hata:
'use strict';
function guvenli() {
  kazara = 42; // ReferenceError: kazara is not defined
}

ES6 modülleri (import/export) otomatik olarak strict modda çalışır. React, Vue, Angular projelerinde bu zaten varsayılandır.

Destructuring Assignment ile let/const

JavaScript 
// Dizi destructuring
const [ilk, ikinci, ...geri_kalan] = [1, 2, 3, 4, 5];

// Nesne destructuring — varsayılan değerle
const { ad, yas, sehir = 'Bilinmiyor' } = kullanici;

// Fonksiyon parametresinde
function selamla({ ad, yas }) {
  console.log(`${ad}, ${yas} yaşında`);
}

JavaScript

// Eski yöntem

// Template literal — temiz ve güçlü

// İfade desteği

Bölüm 02

Hoisting: Motor Kodu Önceden Tarar

Hoisting, JavaScript motorunun kodu çalıştırmadan önce iki geçiş yapmasından kaynaklanır. İlk geçişte tanımlamalar taranır; ikinci geçişte kod satır satır yürütülür.

var Hoisting

var tanımlamaları hoisted olduğunda, değişkenin adı kapsamın tepesine taşınır ama değer taşınmaz:

JavaScript 
console.log(x); // undefined — hata değil!
var x = 5;
console.log(x); // 5

// Motorun gördüğü (kavramsal):
var x;            // tanım yukarı taşındı
console.log(x); // undefined
x = 5;            // atama yerinde kaldı
console.log(x); // 5

Fonksiyon Hoisting

Fonksiyon bildirimleri tamamen hoisted olur; tanımdan önce çağrılabilir. Fonksiyon ifadeleri için bu geçerli değildir:

JavaScript 
selamla(); // "Merhaba!" — tamamen hoisted
function selamla() { console.log("Merhaba!"); }

topla(2, 3); // TypeError: topla is not a function
var topla = function(a, b) { return a + b; };
// var topla; yukarı taşındı (undefined), atama taşınmadı

let ve const — TDZ (Temporal Dead Zone)

let ve const de hoisted olur ama başlangıç değeri atanmaz. Tanım satırına kadar bu değişkenlere erişim ReferenceError üretir:

JavaScript 
console.log(y); // ReferenceError: Cannot access 'y' before initialization
               // (TDZ — y hoisted ama kullanılamıyor)
let y = 42;     // TDZ sona erdi
console.log(y); // 42 ✓
✦
TDZ'nin Avantajı

TDZ bir kısıtlama gibi görünse de var'ın sessiz undefined davranışından çok daha güvenlidir. var hatayı gizler; let/const anında bildirir. Hatayı erken yakalamak, büyük projelerde çok değerlidir.

İyi Pratikler

  1. Değişkenleri kullanmadan önce tanımlayın — var buna izin verse de, önce tanımlamak okunabilirliği artırır.
  2. let/const kullanın — TDZ, hoisting kaynaklı gizli hataları ortadan kaldırır.
  3. 'use strict' etkinleştirin — Bildirilmemiş değişkene atamayı hata olarak yakalar.
  4. Fonksiyon ifadelerini kullanımdan önce tanımlayın — hoisting'e güvenmek okunabilirliği azaltır.

Bölüm 03

Tip Coercion: JavaScript'in Sessiz Dönüşümleri

Tip coercion, farklı tiplerdeki değerler üzerinde işlem yapılırken JavaScript'in otomatik gerçekleştirdiği tip dönüşümüdür. Hata vermez — sadece beklenmedik sonuçlar üretir.

typeof Operatörü

JavaScript 
typeof 42            // "number"
typeof "merhaba"     // "string"
typeof true           // "boolean"
typeof undefined      // "undefined"
typeof {}             // "object"
typeof []             // "object"  ← dizi de object!
typeof function(){}   // "function"
typeof null           // "object"  ← TARİHİN HATASI
⚠
null Kontrolü

typeof null === "object" olduğu için typeof ile null kontrolü yapılamaz. Bunun yerine x === null kullanın. 1995'te null pointer'ın bit temsili 000'dı; nesne etiketi de 000'dı. Bu hata 1997'de fark edildi ama web'i kırmamak için hiç düzeltilmedi.

NaN — Sayı Olmayan Sayı

JavaScript 
parseInt("abc")    // NaN
0 / 0              // NaN
Math.sqrt(-1)      // NaN

typeof NaN         // "number"  ← sürpriz!
NaN === NaN        // false    ← kendine bile eşit değil (IEEE 754)

isNaN("abc")       // true  (önce sayıya çevirir — dikkatli olun)
Number.isNaN(NaN)  // true  ← doğru ve güvenli yol
Number.isNaN("abc") // false ← string değil NaN
⚠
isNaN vs Number.isNaN

isNaN() argümanı önce sayıya çevirir. Bu yüzden isNaN("abc") → true döner. Her zaman Number.isNaN() kullanın.

+ Operatörünün İkili Kişiliği

JavaScript 
1 + 2 + "3"    // "33"  (önce 1+2=3, sonra 3+"3"="33")
"1" + 2 + 3    // "123" (önce "1"+2="12", sonra "12"+3="123")
"5" - 2         // 3    ("5" → 5, sonra 5-2)
"abc" - 2       // NaN

// Güvenli dönüşümler:
Number("42")      // 42  — tercih edilen
parseInt("42px")  // 42  — sayısal başlangıçlı string
String(42)        // "42"
`${42}`           // "42"  — template literal

== vs === — Gevşek ve Katı Eşitlik

JavaScript 
// == ile sürprizler (tip dönüşümü yapar):
0 == false         // true  ← !
"" == false        // true  ← !
null == undefined  // true  (özel kural)
"1" == 1           // true  ← !
[] == false        // true  ← !!

// === ile her şey açık (tip dönüşümü yok):
0 === false        // false — farklı tip
"1" === 1          // false — farklı tip
1 === 1            // true  ✓
null === null      // true  ✓
✕
Altın Kural

Her zaman === kullanın. ESLint eqeqeq kuralı modern projelerde =='yi yasaklar. Tek meşru == kullanımı: x == null (hem null hem undefined'ı tek seferde yakalar).

== Algoritması: Adım Adım

ECMA-262 Abstract Equality Comparison algoritması zincirleme 5 kural uygular. [] == false örneği:

Adım adım 
[] == false
// Adım 1: false boolean → sayıya çevir → 0
[] == 0
// Adım 2: [] nesne → primitive → [].toString() → ""
"" == 0
// Adım 3: "" string → sayıya çevir → Number("") → 0
0 == 0
// Adım 4: her ikisi number → === ile karşılaştır
// true!

Form Değerleri

JavaScript 
// 6 falsy değer — sadece bunlar:

Bölüm 04

Kısa Devre Operatörleri: &&, ||, ??

C#/Java'da mantıksal operatörler true/false döndürür. JavaScript'te ise operandların kendisini döndürürler. Bu fark hem güçlü kalıplara hem de tuzaklara yol açar.

&& — AND Operatörü

İlk falsy değeri döndürür; tümü truthy ise son değeri döndürür:

JavaScript 
true && "merhaba"    // "merhaba"
false && kabul()    // false — kabul() çağrılmaz!
null && "bir şey"   // null
1 && 2 && 3         // 3 — hepsi truthy

// Pratik: koşullu işlem
isLoggedIn && showDashboard();
user?.getProfile?.();

|| — OR Operatörü

İlk truthy değeri döndürür; tümü falsy ise son değeri döndürür:

JavaScript 
null || "varsayılan"   // "varsayılan"
"değer" || "var."    // "değer"
0 || false || 42    // 42 — ilk truthy

// Varsayılan değer
const ad   = isim || "Misafir";
const port = config.port || 3000;
⚠
|| ile 0 Tuzağı

||, 0 ve "" gibi geçerli ama falsy değerleri de varsayılana düşürür. config.port = 0 geçerli bir port numarasıdır ama port || 3000 → 3000 döner. Bu durumda ?? kullanın.

?? — Nullish Coalescing (ES2020)

Yalnızca sol taraf null veya undefined ise sağ tarafı döndürür. 0, "", false için tetiklenmez:

JavaScript 
null ?? "varsayılan"       // "varsayılan"
0    ?? "varsayılan"       // 0    ← || ile fark burada!
""   ?? "varsayılan"       // ""   ← || ile fark burada!
false ?? "varsayılan"      // false

// Sayfa numarası 0 olabilir:
const sayfa = response.page ?? 1;    // ✓ 0 korunur
const yanlis = response.page || 1;   // ✗ 0 → 1'e düşer!

Optional Chaining (?.) — ES2020

Nesne zincirinde herhangi bir adım null/undefined ise hata vermek yerine undefined döndürür:

JavaScript 
// Eski yöntem:
const sehir = user && user.adres && user.adres.sehir;

// ?. ile temiz:
const sehir = user?.adres?.sehir;

// ?? ile birlikte:
const sehir = user?.adres?.sehir ?? "Bilinmiyor";

// Metot ve dizi erişimi:
user?.getAdres?.();
arr?.[0]?.ad;

Atama Operatörleri: ??=, ||=, &&= (ES2021)

JavaScript 
x ??= "varsayılan";   // sadece null/undefined ise atar
x ||= "varsayılan";   // falsy ise atar
x &&= x.toUpperCase(); // truthy ise atar

// Pratik:
options ??= {};
options.timeout ??= 5000;

Guard Clause Kalıbı

JavaScript 
// İç içe if — okunması zor:
function siparisIsle(kullanici) {
  if (kullanici) { if (kullanici.sepeti) { /* ... */ } }
}

// Guard clause — temiz:
function siparisIsle(kullanici) {
  if (!kullanici) return;
  if (!kullanici.sepeti) return;
  if (kullanici.sepeti.length === 0) return;
  // buraya geldiysek her şey hazır
}

Async Kod ve Operatör Önceliği

JavaScript 
async function kullaniciGetir(id) {
  const k = await api.getKullanici(id);
  return {
    ad:        k?.ad ?? 'Anonim',
    tercihler: k?.tercihler ?? {},
  };
}

// ?? + || birleştirme — PARANTEZ ZORUNLU:
const x = a || b ?? c;  // SyntaxError!
const x = (a || b) ?? c; // ✓

}
console.log(a); // (3) ?
console.log(b); // (4) ?

// Cevaplar: (1) 10  (2) 20  (3) 10  (4) 2

Egzersiz

console.log(typeof y); // (2) ?  → ReferenceError

3.1 — Coercion Sonuçları

Egzersiz

Sık Yapılan Hatalar

Hata 1: var ile Closure Tuzağı

JavaScript 
// HATA — hepsi 3 yazar
for (var i=0; i<3; i++) setTimeout(() => console.log(i), 100);
// Çıktı: 3, 3, 3

// ÇÖZÜM — let ile her iterasyon kendi kapsamına sahip
for (let i=0; i<3; i++) setTimeout(() => console.log(i), 100);
// Çıktı: 0, 1, 2

Hata 2: == ile Beklenmedik Eşitlik

JavaScript 
const kullanici = getir(); // null dönebilir
if (kullanici == false) { } // null==false → false! (beklenmedik)

// ÇÖZÜM
if (kullanici == null) { } // ✓ null ve undefined'ı yakalar

Hata 3: NaN Kontrolü

JavaScript 
if (sayi === NaN)           // YANLIŞ — NaN hiçbir şeye eşit değil
if (isNaN(sayi))            // YANLIŞ — string'i dönüştürür
if (Number.isNaN(sayi))     // DOĞRU ✓
if (!Number.isFinite(sayi)) // DOĞRU ✓ (NaN + Infinity yakalar)

Hata 4: || ile 0 Kaybı

JavaScript 
function stokGuncelle(miktar) {
  const m = miktar || 1;  // HATALI — 0 gelirse 1 olur!
  const m = miktar ?? 1;  // DOĞRU  — 0 korunur ✓
}

  var urun = { ad, fiyat, stok: stok || 0 }; // stok=0 kaybolur!
  urunler.push(urun);

for (var i=0; i<urunler.length; i++) {

}
DÜZELTİLMİŞ KOD 
const urunler = [];

function urunEkle(ad, fiyat, stok) {
  const urun = {
    ad,
    fiyat: Number(fiyat),   // açık dönüşüm
    stok:  stok ?? 0,       // 0 korunur ✓
  };
  urunler.push(urun);
}

const urunEkle2 = (urun, adet = 1) => {
  const idx = urunler.findIndex(u => u.id === urun.id); // ===
  idx !== -1 ? urunler[idx].adet += adet : urunler.push({ ...urun, adet });
};

6.1 — null/undefined Karşılaştırmaları

Egzersiz 
null == 0           // false — sadece null/undef ile eşit
null == false       // false — aynı kural
null == undefined   // true  — özel kural ✓
undefined == false  // false — (false→0, undef→NaN, NaN≠0)

6.2 — Hoisting + Shadow

Egzersiz 
let x = 'global';

function test() {
  console.log(x); // (1) ReferenceError! — iç let TDZ'de
  let x = 'lokal';
  console.log(x); // (2) "lokal"
}

test();
console.log(x); // (3) "global" — dış kapsam değişmedi
