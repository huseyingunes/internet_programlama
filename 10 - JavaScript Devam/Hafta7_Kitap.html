Bölüm 01

this — Bağlama Göre Değişen Kimlik

C# ve Java'da this her zaman tanımlandığı nesneyi gösterir. JavaScript'te ise this'in değeri, fonksiyonun nasıl çağrıldığına göre çalışma zamanında belirlenir.

📜
Tarihsel Bağlam

Brendan Eich, JavaScript'i 1995'te 10 günde tasarlarken this davranışını bilinçli olarak Java'dan farklı bıraktı. Amaç: hem nesne-yönelimli hem fonksiyonel programlamayı destekleyen esnek yapı. Bu esneklik yıllar içinde dilin en çok şikâyet edilen özelliği oldu. ES6'da arrow function bu sorunu kısmen, ES2022'de class field syntax ise büyük ölçüde çözdü.

Global Bağlam

Herhangi bir nesne bağlamı olmadan çağrılan fonksiyonlarda this tarayıcıda window, Node.js'te global, strict mode'da undefined'dır:

JavaScript
// Tarayıcı — sloppy mode
console.log(this === window); // true

function normal() {
  console.log(this); // Window
}
normal();

// Strict mode:
'use strict';
function guvenli() {
  console.log(this); // undefined
}
guvenli();

Method Bağlamı — Noktanın Soluna Bak

Bir nesnenin metodu olarak çağrılan fonksiyonda this, noktanın solundaki nesneyi işaret eder. Bu kural mekanik ve kesindir:

JavaScript
const kullanici = {
  ad: 'Ahmet',
  yas: 25,
  selamla() {
    console.log(`Merhaba, ${ this.ad}!`);
  },
  yasArtir() {
    this.yas++;
    return this; // method chaining
  }
};

kullanici.selamla();  // "Merhaba, Ahmet!" — this = kullanici
kullanici.yasArtir().yasArtir(); // zincirleme

// Dikkat: metodu "koparmak"
const fn = kullanici.selamla;
fn(); // undefined veya TypeError — this kayıyor!
✕
Method Kopyalama Tuzağı

Bir metodu değişkene atadığınızda veya callback olarak geçirdiğinizde this bağlamı kaybolur. Bu, JavaScript'teki en yaygın hata kaynaklarından biridir. Çözüm: bind(), arrow function wrapper veya arrow field sözdizimi.

Constructor Bağlamı — new Anahtar Kelimesi

new ile çağrıldığında JS motoru şu adımları izler: boş nesne oluştur → fonksiyonun prototype'ını bağla → this'i yeni nesneye bağla → fonksiyonu çalıştır → nesneyi döndür:

JavaScript
function Araba(marka, model, yil) {
  this.marka = marka;
  this.model = model;
  this.yil   = yil;
  this.tanitim = function() {
    return `${ this.yil} ${ this.marka} ${ this.model}`;
  };
}

const a1 = new Araba('Toyota', 'Corolla', 2022);
const a2 = new Araba('BMW', '3 Serisi', 2023);

a1.tanitim(); // "2022 Toyota Corolla"
a2.tanitim(); // "2023 BMW 3 Serisi"

// new olmadan çağrılırsa felaket:
const a3 = Araba('Honda', 'Civic', 2021);
// marka, model, yil global değişken olur!

Event Handler Bağlamı

JavaScript
const btn = document.getElementById('gonder');

// function: this = btn elementi ✓
btn.addEventListener('click', function() {
  console.log(this.id); // 'gonder'
  this.disabled = true;
});

// arrow: this = dış scope (dikkat!)
btn.addEventListener('click', () => {
  console.log(this); // Window veya undefined
});

call, apply ve bind — Manuel Bağlama

JavaScript
function selamla(mesaj, noktalama) {
  return `${mesaj}, ${ this.ad}${noktalama}`;
}
const kisi = { ad: 'Zeynep' };

// call: hemen çalışır, argümanlar tek tek
selamla.call(kisi, 'Merhaba', '!'); // "Merhaba, Zeynep!"

// apply: hemen çalışır, argümanlar dizi
selamla.apply(kisi, ['Merhaba', '!']); // "Merhaba, Zeynep!"

// bind: yeni fonksiyon döner, this sabitlenir
const merhaba = selamla.bind(kisi, 'Merhaba');
merhaba('!');  // "Merhaba, Zeynep!"
merhaba('...'); // "Merhaba, Zeynep..."

this Kayması ve 3 Çözüm Yolu

JavaScript
const zamanlayici = {
  saniye: 0,
  baslat() {
    setInterval(function() {
      this.saniye++; // this = Window, kayma!
    }, 1000);
  }
};

// Çözüm 1: Arrow function (önerilen)
setInterval(() => { this.saniye++; }, 1000);

// Çözüm 2: bind()
setInterval((function() {
  this.saniye++;
}).bind(this), 1000);

// Çözüm 3: const self = this (eski yöntem)
const self = this;
setInterval(function() { self.saniye++; }, 1000);
Bölüm 02

Closure — Fonksiyon Doğduğu Yeri Hatırlar

Closure, bir fonksiyonun tanımlandığı scope'taki değişkenlere — o scope sona erse bile — erişebilmesidir. Lexical scope ile garbage collection'ın birleşiminden doğar.

📜
Tarihsel Kök

1968 — Peter Landin, SECD makinesinde closure'ı ilk kez formüle etti. 1975 — Sussman & Steele, Scheme dilinde closure'ı ana akıma taşıdı. 1995 — Brendan Eich, Scheme'den doğrudan ilham alarak closure'ı JavaScript'in çekirdeğine koydu. Bugün Python, Ruby, Swift, Kotlin ve Java lambda'larında da closure mevcuttur.

Lexical Scope

JavaScript'te kapsam, kodu yazdığınız yere göre belirlenir — çalıştığınız yere göre değil:

JavaScript
const x = 'global';

function dis() {
  const y = 'dis';

  function ic() {
    const z = 'ic';
    console.log(x); // 'global' — dışarıdan
    console.log(y); // 'dis'    — üst scope
    console.log(z); // 'ic'     — kendi
  }
  ic();
}

dis();
// console.log(y) → ReferenceError — erişilemez

Closure Mekanizması

JavaScript
function disKapsam() {
  let sayac = 0;
  // sayac, closure tarafından yaşatılacak

  function icFonksiyon() {
    sayac++;
    return sayac;
  }

  return icFonksiyon; // fonksiyonu dışa aktar
}

const artir = disKapsam();
// disKapsam() bitti — ama sayac bellekte yaşıyor!

console.log(artir()); // 1
console.log(artir()); // 2
console.log(artir()); // 3
// Her çağrıda aynı sayac güncellenir — closure!
⚙
Neden Bellekte Kalıyor?

JS motoru her fonksiyon çağrısı için bir "execution context" oluşturur. Bu context bir "environment record" (değişken tablosu) içerir. icFonksiyon, disKapsam'ın environment record'una referans tutar. Bu referans var olduğu sürece garbage collector o record'u serbest bırakamaz.

Private Değişken Simülasyonu

JavaScript
function sayacOlustur(baslangic = 0) {
  let _deger = baslangic; // dışarıdan erişilemez
  let _adim  = 1;

  return {
    artir()         { _deger += _adim; return this; },
    azalt()         { _deger -= _adim; return this; },
    sifirla()       { _deger = baslangic; return this; },
    adimAyarla(n)   { _adim = n; return this; },
    deger()         { return _deger; },
  };
}

const s = sayacOlustur(10);
s.adimAyarla(5).artir().artir(); // zincirleme
s.deger();   // 20
s._deger;    // undefined — gizli ✓

Fonksiyon Fabrikası

JavaScript
function katsayiUret(n) {
  return (x) => x * n; // n bu closure'a ait
}

const ikiyle = katsayiUret(2); // n=2 kalıcı
const ucle   = katsayiUret(3); // n=3 kalıcı
const kdv    = katsayiUret(1.18);

ikiyle(5);  // 10
ucle(5);    // 15
kdv(100);  // 118
// Her closure kendi n'ini saklar
✦
React Bağlantısı

React'taki useState, useReducer, useCallback ve useMemo hook'larının tamamı closure üzerine kuruludur. Her render yeni bir closure oluşturur; state değerlerini renderlar arasında taşıyan mekanizma budur.

Memoization

JavaScript
function memoize(fn) {
  const onbellek = {}; // closure ile gizlenir

  return function(...args) {
    const key = JSON.stringify(args);
    if (key in onbellek) return onbellek[key];
    onbellek[key] = fn(...args);
    return onbellek[key];
  };
}

const memoFib = (function() {
  const cache = { 0: 0, 1: 1 }; // IIFE ile gizli
  return function fib(n) {
    if (n in cache) return cache[n];
    cache[n] = fib(n-1) + fib(n-2);
    return cache[n];
  };
})(); // IIFE — hemen çalışır

memoFib(40); // Anlık

Bölüm 03

Arrow Function — Sadece Kısaltma Değil

Arrow function'ın function'dan temel farkı this bağlamıdır. Sözdizimi kısalığı sadece yan fayda.

Sözdizimi

JavaScript
// Geleneksel
function topla(a, b) { return a + b; }

// Arrow — implicit return
const topla = (a, b) => a + b;

// Tek parametre, parantez opsiyonel
const karesi = x => x * x;

// Parametresiz
const selamla = () => 'Merhaba';

// Nesne döndürmek — parantez şart!
const nesne = x => ({ deger: x, iki: x * 2 });

this Farkı — Lexical this

Arrow function kendi this bağlamını oluşturmaz; tanımlandığı scope'un this'ini kullanır (lexical this):

✕ function — dinamik this
const obj = {
  ad: 'Ali',
  fn: function() {
    return this.ad;
    // this = obj ✓
  }
};
obj.fn(); // 'Ali'
⚠ arrow — lexical this
const obj = {
  ad: 'Ali',
  arrow: () => {
    return this.ad;
    // this = global!
  }
};
obj.arrow(); // undefined

Ne Zaman Hangisi?

  • Object method: function — this = nesne olmalı
  • Constructor: function veya class — arrow new ile kullanılamaz
  • setTimeout/setInterval içinde this: Arrow — dış this'i yakalar
  • Array.map/filter/reduce: Arrow — kısa, this gerekmez
  • Event handler (this = element): function
Bölüm 04

Callback, Event Loop ve Asenkron JS

JavaScript'in asenkron programlama modeli tek thread üzerinde nasıl çalıştığını açıklar. Bu modelin temeli callback, motoru ise Event Loop'dur.

Callback Deseni

Callback: başka bir fonksiyona argüman olarak geçirilen, belirli bir olay gerçekleştiğinde çağrılmak üzere bırakılan fonksiyon:

JavaScript
function isle(veri, callback) {
  const sonuc = veri.toUpperCase();
  callback(sonuc);
}

isle('merhaba', (s) => console.log(s)); // 'MERHABA'

// DOM olayı:
btn.addEventListener('click', () => console.log('Tıklandı'));

// Zamanlayıcı:
setTimeout(() => console.log('1 saniye'), 1000);

// Array metodları:
[1,2,3].filter(x => x % 2 === 0); // [2]
📜
Callback'in Kökeni

Callback deseni 1960'lardaki continuation-passing style (CPS) programlama stiliyle kökten bağlantılıdır. UNIX sinyalleri, Smalltalk-80 event handler'ları, Scheme continuation'ları — hepsi aynı fikrin farklı yansımasıdır. Node.js (2009) bu deseni sunucu tarafında ana akıma taşıdı.

Neden Tek Thread?

  • DOM'da race condition yok: Birden fazla thread DOM'u aynı anda değiştirirse tutarsızlık kaçınılmaz olur.
  • Basit programlama modeli: Mutex, semaphore, monitor gibi eşzamanlılık primitiflerine gerek yok.
  • I/O verimliliği: Async model, I/O-yoğun iş yükleri için thread-per-request modelinden çok daha verimlidir.

Event Loop Modeli

⚙
Bileşenler

Call Stack: Fonksiyon çağrıları yığını (LIFO). Web APIs: setTimeout, fetch, addEventListener — JS değil ortam yönetir. Microtask Queue: Promise.then(), queueMicrotask() — stack boşalınca öncelikli. Task Queue (Macro): setTimeout, setInterval — Microtask boşaldıktan sonra. Event Loop: Stack boş → önce Microtask → sonra Task → tekrar.

JavaScript — Sıra Tahmini
console.log('A');

setTimeout(() => console.log('B — setTimeout 0ms'), 0);

Promise.resolve().then(() => console.log('C — Promise'));

queueMicrotask(() => console.log('D — queueMicrotask'));

console.log('E');

// Çıktı: A → E → C → D → B
// A, E: synchronous (Call Stack)
// C, D: Microtask Queue (önce)
// B:    Task Queue (en son)
⚠
setTimeout(fn, 0) Yanılgısı

"0ms bekle" = "hemen çalış" değildir. "En az 0ms bekle, Task Queue'ya ekle" demektir. Microtask Queue her zaman Task Queue'dan önce işlenir. Bu performans optimizasyonlarında kritik bir detaydır.

Callback Hell

Callback Hell — Kaçınılacak
kullaniciyiGetir(userId, (kullanici, hata) => {
  if (hata) { hataYonet(hata); return; }

  siparisleriGetir(kullanici.id, (siparisler, hata) => {
    if (hata) { hataYonet(hata); return; }

    urunleriGetir(siparisler[0].id, (urunler, hata) => {
      if (hata) { hataYonet(hata); return; }
      goster(kullanici, siparisler, urunler);
      // Her hata kolu tekrar, test imkânsız
    });
  });
});

Promise

JavaScript
// Promise zinciri — doğrusal, tek catch
kullaniciyiGetir(userId)
  .then(k => siparisleriGetir(k.id))
  .then(s => urunleriGetir(s[0].id))
  .then(goster)
  .catch(hataYonet);

// Paralel istekler:
Promise.all([
  kullaniciyiGetir(userId),
  ayarlariGetir(userId),
]).then(([kullanici, ayarlar]) => goster(kullanici, ayarlar));

async/await

JavaScript
async function yukle(userId) {
  try {
    const kullanici  = await kullaniciyiGetir(userId);
    const siparisler = await siparisleriGetir(kullanici.id);
    goster(kullanici, siparisler);
  } catch (hata) {
    hataYonet(hata);
  }
}

// Paralel — Promise.all ile birlikte kullanın:
async function paralel(userId) {
  const [k, a] = await Promise.all([
    kullaniciyiGetir(userId),
    ayarlariGetir(userId),
  ]);
  goster(k, a);
}
ℹ
async/await'in Gerçeği

async/await arka planda Promise kullanır. Bir await gördüğünde JS motoru kontrolü Event Loop'a bırakır (bloklamaz). Promise çözümlendiğinde devam Microtask Queue'ya eklenir. Yani "senkron görünümlü asenkron" kod yazar — gerçekte bloklamaz.

Pratik Egzersizler ve Sık Hatalar

Teoriyi pekiştirmenin en iyi yolu uygulamalı denemedir.

this Egzersizleri

Egzersiz 1.1
const a = { x: 10, getX() { return this.x; } };
const b = { x: 20 };

a.getX();               // (1) → 10
b.getX = a.getX; b.getX(); // (2) → 20
const fn = a.getX;
fn();                   // (3) → undefined (global)
fn.call({ x: 30 });    // (4) → 30
fn.call(a);             // (5) → 10

Closure Egzersizleri

Banka Hesabı

Egzersiz 2.1
function hesapOlustur(baslangic = 0) {
  let _bakiye = baslangic;

  return {
    yatir(m)  { if (m <= 0) return 'Geçersiz'; _bakiye += m; return _bakiye; },

const h = hesapOlustur(1000);

JavaScript — Stale Closure

API Servisi — this + Arrow + Closure

JavaScript
class ApiServis {
  constructor(baseUrl) {
    this.baseUrl = baseUrl;
    this.token   = null;
  }

  // Arrow field: this daima örneği gösterir
  girisYap = async (kullanici, sifre) => {
    const yanit = await fetch(this.baseUrl + '/login', {
      method: 'POST',
      body: JSON.stringify({ kullanici, sifre }),
    });
    const veri = await yanit.json();
    this.token = veri.token;
    return veri;
  };

  // Closure: token dışa sızdırılmadan header ekle
  istekYap = async (endpoint) => {
    return fetch(this.baseUrl + endpoint, {
      headers: { Authorization: 'Bearer ' + this.token }
    }).then(r => r.json());
  };
}

Tam Sistem Egzersizi

JavaScript — Closure + Arrow + async
function kullaniciModulu() {
  const _liste = []; // private closure

  const _apidanCek = (id) => new Promise(resolve =>
    setTimeout(() => resolve({ id, ad: 'User' + id }), 100)
  );

  return {
    async yukle(idler, callback) {
      const sonuclar = await Promise.all(
        idler.map(id => _apidanCek(id))
      );
      sonuclar.forEach(u => _liste.push(u));
      callback?.(sonuclar); // opsiyonel callback
      return sonuclar;
    },
    liste: () => [..._liste],
    temizle: () => _liste.splice(0),
  };
}

const m = kullaniciModulu();
await m.yukle([1,2,3], s => console.log('Yüklendi:', s.length));
m._liste; // undefined — gizli ✓
