Bölüm 01

Neden Promise ve async/await?

Hafta 7'de callback'leri ve Event Loop'u öğrendik. Asenkron işlemlerin neden gerekli olduğunu biliyoruz. Ama callback'lerin sınırına geldik — buradan modern bir çözüme geçiyoruz.

Callback Hell'i Hatırlayalım

Üç API çağrısının birbirine bağımlı olduğu bir senaryoda callback'ler iç içe geçer:

JavaScript — Eski Yöntem
kullaniciyiGetir(id, (kullanici, hata) => {
  if (hata) { hataGoster(hata); return; }

  siparisleriGetir(kullanici.id, (siparisler, hata) => {
    if (hata) { hataGoster(hata); return; }

    urunleriGetir(siparisler[0].id, (urunler, hata) => {
      if (hata) { hataGoster(hata); return; }
      ekrandaGoster(kullanici, siparisler, urunler);
    });
  });
});
  • Görsel piramidleşme: her yeni adım sağa kayıyor, kod okunmuyor.
  • Hata yönetimi tekrarı: her seviyede aynı if (hata) kontrolü.
  • Akış takibi zor: kod yukarıdan aşağıya akmıyor, içe doğru gidiyor.
  • Test zorluğu: iç içe callback'leri mock'lamak karmaşık.
📜
Promise'in Tarihçesi

Promise kavramı 1976'da MIT'de Daniel Friedman ve David Wise tarafından geliştirildi. Argus dilinde (1988) ve E dilinde (1997) yaygınlaştı. JavaScript'e gelmesi uzun sürdü: 2010'larda jQuery'nin Deferred objesi ve Q kütüphanesi öncülük etti. ES6 (2015) ile dil seviyesinde standartlaştı. async/await ise 2012'de C# 5.0'da popülerleşti; 2017'de JavaScript'e ES2017 standardı olarak kazandırıldı.

Promise ile Aynı Kod

Promise zinciri
kullaniciyiGetir(id)
  .then(kullanici => siparisleriGetir(kullanici.id))
  .then(siparisler => urunleriGetir(siparisler[0].id))
  .then(urunler => ekrandaGoster(urunler))
  .catch(hata => hataGoster(hata));
// Tek catch — tüm zincirdeki hataları yakalar
async/await
async function yukle(id) {
  try {
    const kullanici  = await kullaniciyiGetir(id);
    const siparisler = await siparisleriGetir(kullanici.id);
    const urunler    = await urunleriGetir(siparisler[0].id);
    ekrandaGoster(urunler);
  } catch (hata) {
    hataGoster(hata);
  }
}
✦
Bu Bölümde Ne Öğreneceğiz?

Promise'in üç durumunu, zincirleme kullanımını, async/await sözdizimini, Promise.all gibi kombinatörleri ve sık yapılan hataları öğreneceğiz. Bölüm sonunda PokéAPI ile gerçek bir Pokemon Kartları uygulaması yapacağız — adım adım, derste birlikte yazılabilecek şekilde.

Bölüm 02

Promise Temelleri

Promise, asenkron bir işlemin nihai sonucunu (ya da hatasını) temsil eden bir nesnedir. "Şu an cevabım yok ama gelecekte ya değer döndüreceğim ya da hata fırlatacağım" sözü gibi düşünün.

Üç Durum

Bir Promise nesnesinin üç durumu vardır:

pending
Beklemede
İşlem devam ediyor
fulfilled
Başarılı
Bir değer döndürdü
rejected
Reddedildi
Bir hata oluştu

Bir Promise pending'den fulfilled veya rejected durumuna geçer. Bu geçiş tek yönlüdür — fulfilled olduktan sonra rejected olamaz, ya da tersi.

.then() · .catch() · .finally()

JavaScript
// fetch() bir Promise döner
fetch('https://api.example.com/data')
  .then(yanit => {
    console.log('Yanıt geldi:', yanit.status);
    return yanit.json();   // Bu da bir Promise döner
  })
  .then(veri => {
    console.log('Veri:', veri);
  })
  .catch(hata => {
    console.error('Bir şey ters gitti:', hata);
  })
  .finally(() => {
    console.log('İster başarılı, ister başarısız — bu çalışır');
  });
ℹ
then, catch, finally

.then(onFulfilled) — Promise başarılı olursa çalışır. .catch(onRejected) — Promise hata verirse çalışır. .finally(callback) — sonuç ne olursa olsun çalışır (loading state'i kapatmak için ideal).

Promise Zinciri

.then() içinde yeni bir Promise döndürürseniz, zincir devam eder. Her .then() bir sonrakine değer geçirir:

JavaScript
fetch('/api/kullanici/1')
  .then(yanit => yanit.json())          // Promise döner
  .then(kullanici => {
    console.log('Kullanıcı:', kullanici.ad);
    return fetch('/api/siparisler/' + kullanici.id);
  })                                     // Promise döner
  .then(yanit => yanit.json())
  .then(siparisler => {
    console.log('Siparişler:', siparisler.length);
  })
  .catch(hata => console.error(hata));
// Zincirin herhangi bir yerindeki hata tek catch'e düşer

Promise Oluşturmak

Çoğu zaman hazır Promise'leri tüketirsiniz. Ama kendi Promise'inizi yaratmanız gerekirse:

JavaScript
function gecikme(ms) {
  return new Promise((resolve, reject) => {
    if (ms < 0) {
      reject(new Error('Negatif süre olamaz'));
      return;
    }
    setTimeout(() => {
      resolve('Tamam, ' + ms + 'ms bekledim');
    }, ms);
  });
}

gecikme(1000)
  .then(mesaj => console.log(mesaj))   // 'Tamam, 1000ms bekledim'
  .catch(hata => console.error(hata));

gecikme(-100)
  .catch(hata => console.error(hata.message));  // 'Negatif süre olamaz'
  • resolve(deger) — Promise'i fulfilled durumuna geçirir, .then()'e değer iletir.
  • reject(hata) — Promise'i rejected durumuna geçirir, .catch()'e hata iletir.
Bölüm 03

async/await — Senkron Görünümlü Asenkron

ES2017 ile gelen async/await sözdizimi, Promise tabanlı kodu senkron kod gibi yazmamızı sağlar.

async Fonksiyon

Bir fonksiyonun önüne async yazarsanız iki şey olur:

  1. Fonksiyon her zaman bir Promise döndürür — siz ne döndürürseniz döndürün, otomatik olarak Promise'e sarılır.
  2. Fonksiyon içinde await kullanabilirsiniz.
JavaScript
// async olmayan normal fonksiyon
function selamla() { return 'Merhaba'; }
selamla(); // 'Merhaba' — düz string

// async fonksiyon
async function asyncSelamla() { return 'Merhaba'; }
asyncSelamla(); // Promise { 'Merhaba' } — Promise!
asyncSelamla().then(s => console.log(s)); // 'Merhaba'

await Anahtar Kelimesi

await, bir Promise'in çözülmesini bekler ve Promise'in değerini döndürür. Promise hata verirse, await hata fırlatır (try/catch ile yakalayabilirsiniz).

JavaScript
// Promise zinciri ile
function eskiUsul() {
  return fetch('/api/users/1')
    .then(r => r.json())
    .then(u => u.name);
}

// async/await ile — aynı işlem
async function modernUsul() {
  const yanit = await fetch('/api/users/1');
  const kullanici = await yanit.json();
  return kullanici.name;
}

// İkisi de aynı şekilde kullanılır:
modernUsul().then(ad => console.log(ad));
ℹ
await Sadece async İçinde

await yalnızca async fonksiyon içinde kullanılabilir. Modül üst seviyesinde (top-level await) ES2022'den sonra destekleniyor ama sınıf metodları ve normal fonksiyonlarda async olmadan kullanılamaz. SyntaxError alırsanız ilk kontrol edilecek şey budur.

Hata Yönetimi: try / catch

JavaScript
async function veriGetir(id) {
  try {
    const yanit = await fetch('/api/users/' + id);
    if (!yanit.ok) {
      throw new Error('HTTP ' + yanit.status);
    }
    const kullanici = await yanit.json();
    return kullanici;
  } catch (hata) {
    console.error('Yükleme başarısız:', hata.message);
    return null;
  } finally {
    console.log('Yükleme bitti');
  }
}
⚠
await Unutmaya Dikkat

await yazmayı unutursanız, Promise nesnesinin kendisi değişkene atanır — gerçek değer değil. const veri = fetch(...) → veri Promise. const veri = await fetch(...) → veri Response objesi. Bu en yaygın yeni başlayan hatasıdır.

Sıralı vs Paralel Bekleme

await'i nasıl kullandığınız performansı dramatik etkiler:

YAVAŞ — Sıralı
async function yavasYukle() {
  const kullanici  = await fetch('/api/users/1').then(r => r.json());
  const siparis    = await fetch('/api/orders').then(r => r.json());
  const urunler    = await fetch('/api/products').then(r => r.json());
  return { kullanici, siparis, urunler };
}
// 3 istek × 100ms = 300ms toplam
HIZLI — Paralel
async function hizliYukle() {
  const [kullanici, siparis, urunler] = await Promise.all([
    fetch('/api/users/1').then(r => r.json()),
    fetch('/api/orders').then(r => r.json()),
    fetch('/api/products').then(r => r.json()),
  ]);
  return { kullanici, siparis, urunler };
}
// 3 istek paralel × max 100ms = 100ms toplam
✦
Sıralı mı Paralel mi?

Bir istek diğerinin sonucuna ihtiyaç duyuyorsa sıralı await zorunludur (önce kullanıcıyı çek, sonra ID'siyle siparişlerini çek). Bağımsız isteklerde Promise.all kullanın — performans 3-5 kat artar.

Bölüm 04

Promise Kombinatörleri

Birden fazla Promise'i bir arada yönetmek için JavaScript bize dört statik metod sunar. Her birinin davranışı farklı, hangisini ne zaman kullanacağınızı bilmek kritik.

Promise.all — Hepsi Başarılı Olmalı

Tüm Promise'leri bekler. Hepsi başarılı olursa sonuçların dizisini döndürür. Biri başarısız olursa tümü reject edilir ("all or nothing"):

JavaScript
const [kullanici, siparisler, mesajlar] = await Promise.all([
  fetch('/api/users/1').then(r => r.json()),
  fetch('/api/orders').then(r => r.json()),
  fetch('/api/messages').then(r => r.json()),
]);

// Sonuçlar dizi olarak gelir — sıra korunur
// Herhangi biri hata verirse hepsi reject olur:
try {
  const sonuclar = await Promise.all([...]);
} catch (hata) {
  // İlk reject'in hatası buraya düşer
}

Promise.allSettled — Hepsini Bekle

Tüm Promise'leri bekler ama hata olsa bile reject olmaz. Her birinin sonucunu ayrı ayrı verir:

JavaScript
const sonuclar = await Promise.allSettled([
  fetch('/api/users'),
  fetch('/api/products'),
  fetch('/api/messages'),
]);

sonuclar.forEach((sonuc, i) => {
  if (sonuc.status === 'fulfilled') {
    console.log('Servis ' + i + ' OK:', sonuc.value);
  } else {
    console.warn('Servis ' + i + ' başarısız:', sonuc.reason);
  }
});

// Sonuç şekli:
// [{ status: 'fulfilled', value: ... },
//  { status: 'rejected', reason: Error },
//  { status: 'fulfilled', value: ... }]

Promise.race — İlk Biten Kazanır

Herhangi bir Promise (fulfilled veya rejected) tamamlandığında o sonucu döndürür. Timeout deseni için yaygındır:

JavaScript
function zamanasimi(promise, ms) {
  const zamanasimi = new Promise((_, reject) =>
    setTimeout(() => reject(new Error('Zaman aşımı')), ms)
  );
  return Promise.race([promise, zamanasimi]);
}

try {
  const veri = await zamanasimi(
    fetch('/api/yavas-endpoint').then(r => r.json()),
    5000
  );
} catch (e) {
  // 'Zaman aşımı' veya gerçek fetch hatası
}

Promise.any — İlk Başarılı Olan

İlk başarılı Promise'i döndürür. Tümü başarısız olursa AggregateError fırlatır:

JavaScript
try {
  const veri = await Promise.any([
    fetch('https://cdn1.example.com/file.json'),
    fetch('https://cdn2.example.com/file.json'),
    fetch('https://cdn3.example.com/file.json'),
  ]).then(r => r.json());
} catch (hata) {
  // hata instanceof AggregateError
  // hata.errors → tüm bireysel hatalar dizisi
}

Karşılaştırma Tablosu

MetodTüm sonuçları bekler mi?Hata davranışıKullanım
Promise.allEvetBiri hata → tümü rejectBağımsız zorunlu istekler
Promise.allSettledEvetAsla reject olmazBağımsız opsiyonel istekler
Promise.raceHayır — ilki yeterİlk biten ne iseTimeout, ilk yanıt

Bölüm 05

Sık Yapılan Hatalar

async/await ile en yaygın 3 hata. Bu üçünden kaçındığınızda kodun büyük kısmı sorunsuz çalışır.

await Unutmak

JavaScript
// HATALI
async function ekle(id) {
  const kullanici = fetch('/api/users/' + id).then(r => r.json());
  console.log(kullanici.ad);
  // 'undefined' — kullanici hâlâ Promise objesi, .ad yok
}

// DOĞRU
async function ekle(id) {
  const kullanici = await fetch('/api/users/' + id).then(r => r.json());
  console.log(kullanici.ad);  // ✓
}

forEach + async — Klasik Tuzak

Bir dizinin her elemanı için asenkron işlem yapmak isterseniz, forEach async fonksiyonu beklemez:

JavaScript
// HATALI — forEach async'i beklemez
async function tumunuYukle(idler) {
  idler.forEach(async (id) => {
    const veri = await fetch('/api/' + id).then(r => r.json());
    console.log(veri);
  });
  console.log('Bitti'); // Hemen yazılır, fetch'ler hâlâ devam ediyor
}

// DOĞRU 1 — sıralı: for...of
async function tumunuYukle(idler) {
  for (const id of idler) {
    const veri = await fetch('/api/' + id).then(r => r.json());
    console.log(veri);
  }
}

// DOĞRU 2 — paralel: map + Promise.all
async function tumunuYukle(idler) {
  const sonuclar = await Promise.all(
    idler.map(id => fetch('/api/' + id).then(r => r.json()))
  );
}
✕
forEach + async = Yarış Durumu

forEach async fonksiyonu beklemez. Tüm callback'leri eşzamanlı başlatır ve hemen devam eder. Asenkron iterasyon için for...of (sıralı) veya Promise.all + map (paralel) kullanın.

fetch 404 Reject Etmez

fetch, HTTP hata kodlarında (404, 500) Promise'i reject etmez. Sadece ağ hatalarında reject eder. Bu yüzden yanit.ok kontrolü her zaman gereklidir:

JavaScript
// EKSİK — 404'te bile devam eder
async function kullaniciGetir(id) {
  const yanit = await fetch('/api/users/' + id);
  return yanit.json();  // 404'te bile parse etmeye çalışır
}

// DOĞRU
async function kullaniciGetir(id) {
  const yanit = await fetch('/api/users/' + id);
  if (!yanit.ok) {
    throw new Error('HTTP ' + yanit.status);
  }
  return yanit.json();
}

Tek Pokemon Çekme Fonksiyonu

2async fonksiyon + fetch · ~7 dk

İlk async fonksiyonumuzu yazalım. Aynı zamanda yanit.ok kontrolünü ve hata yönetimini öğreneceğiz. Script bölümüne ekleyin:

JavaScript
// Adım 2: Tek Pokemon getir
async function pokemonGetir(isim) {
  const url = 'https://pokeapi.co/api/v2/pokemon/' + isim.toLowerCase().trim();
  const yanit = await fetch(url);

  if (!yanit.ok) {
    throw new Error(`'${isim}' bulunamadı (HTTP ${yanit.status})`);
  }

  const veri = await yanit.json();
  return {
    id:      veri.id,
    isim:    veri.name,
    resim:   veri.sprites.other['official-artwork'].front_default,
    tipler:  veri.types.map(t => t.type.name),
    hp:      veri.stats[0].base_stat,
    saldiri: veri.stats[1].base_stat,
  };
}

// Test — konsolda çalıştır:
pokemonGetir('pikachu').then(p => console.log(p));
🧪
Test

Sayfayı yenileyin. Konsolu açın (F12). Pikachu'nun bilgilerini görmelisiniz. 'pikachuxyz' gibi geçersiz bir isim deneyin — hatayı görmelisiniz.

Bu adımda öğrenilen: async fonksiyon, await fetch, yanit.ok kontrolü, hata fırlatma (throw new Error), nesne dönüştürme.

Kartı DOM'a Ekleme + Sıralı Yükleme

3DOM + for...of · ~7 dk

Şimdi gelen veriyi ekrana yansıtalım. Önce kart oluşturma fonksiyonu:

JavaScript
// Adım 3.1: Pokemon verisinden kart elementi oluştur
function kartOlustur(pokemon) {
  const kart = document.createElement('div');
  kart.className = 'kart';
  kart.innerHTML = `
    <img src="${pokemon.resim}" alt="${pokemon.isim}">
    <h3>#${pokemon.id} ${pokemon.isim}</h3>
    <p>${pokemon.tipler.join(', ')}</p>
    <p>HP: ${pokemon.hp} | Atak: ${pokemon.saldiri}</p>
  `;
  return kart;
}

Şimdi buton tıklamasını yakalayalım — ilk versiyon, sıralı await ile:

JavaScript — Sıralı (yavaş)
// Adım 3.2: Buton — sıralı versiyon (yavaş!)
const getirBtn = document.getElementById('getir');
const isimlerInput = document.getElementById('isimler');
const kartlarDiv = document.getElementById('kartlar');
const durumDiv = document.getElementById('durum');

getirBtn.addEventListener('click', async () => {
  const isimler = isimlerInput.value.split(',').map(s => s.trim()).filter(Boolean);
  if (isimler.length === 0) return;

  kartlarDiv.innerHTML = '';
  durumDiv.innerHTML = '<p class="yukleniyor">Yükleniyor...</p>';

  // SIRALI — her birini sırayla bekle
  for (const isim of isimler) {
    try {
      const pokemon = await pokemonGetir(isim);
      kartlarDiv.appendChild(kartOlustur(pokemon));
    } catch (hata) {
      console.error(hata);
    }
  }

5 Pokemon × ortalama 300ms = 1.5 saniye toplam. Oysa istekler birbirinden bağımsız — paralel yapılabilir. Bir sonraki adımda bunu Promise.all ile düzelteceğiz.

Paralel Yükleme (Promise.all)

4Promise.all + map · ~6 dk

Sıralı for döngüsünü, Promise.all ile değiştirelim. Tüm Pokemon'lar aynı anda istek atacak:

JavaScript — Paralel
// Adım 4: Paralel versiyon
getirBtn.addEventListener('click', async () => {
  const isimler = isimlerInput.value.split(',').map(s => s.trim()).filter(Boolean);
  if (isimler.length === 0) return;

  kartlarDiv.innerHTML = '';
  durumDiv.innerHTML = '<p class="yukleniyor">Yükleniyor...</p>';

  try {
    // PARALEL — hepsini aynı anda başlat
    const pokemonlar = await Promise.all(
      isimler.map(isim => pokemonGetir(isim))
    );

    // Hepsi geldi, kartları oluştur
    pokemonlar.forEach(p => kartlarDiv.appendChild(kartOlustur(p)));
    durumDiv.innerHTML = `<p>${pokemonlar.length} Pokemon yüklendi</p>`;
  } catch (hata) {
    durumDiv.innerHTML = `<div class="hata">${hata.message}</div>`;
  }
});

⚠
Promise.all Sorunu

Şu an bir Pokemon ismi yanlış olursa TÜM ekran hataya düşüyor. Geçerli olanlar bile gösterilmiyor. Bir sonraki adımda bunu allSettled ile çözeceğiz.

Hataya Toleranslı (Promise.allSettled)

5Promise.allSettled · ~8 dk

Promise.allSettled, her Promise'in sonucunu ayrı ayrı verir. Başarısız olanları kart yerine hata mesajı olarak gösterelim:

JavaScript — Final
// Adım 5: Hataya toleranslı versiyon — FINAL
getirBtn.addEventListener('click', async () => {
  const isimler = isimlerInput.value.split(',').map(s => s.trim()).filter(Boolean);
  if (isimler.length === 0) return;

  kartlarDiv.innerHTML = '';
  durumDiv.innerHTML = '<p class="yukleniyor">Yükleniyor...</p>';

  // Her sonucu ayrı ayrı al — bir başarısız diğerlerini bozmaz
  const sonuclar = await Promise.allSettled(
    isimler.map(isim => pokemonGetir(isim))
  );

  let basariSayisi = 0;
  const hatalar = [];

  sonuclar.forEach((sonuc, i) => {
    if (sonuc.status === 'fulfilled') {
      kartlarDiv.appendChild(kartOlustur(sonuc.value));
      basariSayisi++;
    } else {
      hatalar.push(`'${isimler[i]}': ${sonuc.reason.message}`);
    }
  });

  // Durum mesajı
  let mesaj = `${basariSayisi}/${isimler.length} Pokemon yüklendi`;
  if (hatalar.length > 0) {
    mesaj += `<div class="hata">${hatalar.join('<br>')}</div>`;
  }
  durumDiv.innerHTML = mesaj;
});
